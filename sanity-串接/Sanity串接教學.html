把「穩定上岸系列文章」接上 Sanity

你已經建好 Sanity 專案，接下來分兩件事：① 設定好結構與連線（一次性）② 之後每次在 Sanity 後台寫文章，首頁三個方塊就會自動更新。照著做，大約 20 分鐘完成。

運作原理：你在 Sanity 後台寫文章 → 首頁的 JavaScript 會去 Sanity 的 API 抓最新 3 篇 → 自動長出卡片。網站本身不用每次改程式。

我已經把首頁 index.html 的文章區塊接好了，你只剩下 3 件事要做：建立文章結構、開放連線權限(CORS)、把你的 Project ID 填進去。

1

在 Sanity 建立「文章」結構

讓後台有「標題／分類／封面／內文」這些欄位可以填。

  1. 打開你電腦上的 Sanity 專案資料夾，找到 schemaTypes（有的範本叫 schemas）。
  2. 把本包的 postType.js 複製進這個資料夾。
  3. 打開同資料夾的 index.js，把 postType 加進去（可參考本包的 schemaTypes-index.js 範例）。
  4. 存檔後，在專案資料夾的終端機執行 npm run dev，瀏覽器打開 http://localhost:3333，左側就會出現「文章」。
還沒裝好 Studio？在專案資料夾終端機先跑一次 npm install，再 npm run dev 即可。要讓別人也能上後台，之後用 npx sanity deploy 部署成線上網址。
2

開放網站讀取權限（CORS）

不設這步，網站會抓不到資料（瀏覽器會擋）。

  1. 到 sanity.io/manage → 點你的專案 → 左側 API。
  2. 找到 CORS origins → Add CORS origin。
  3. 網址填你的網站網域，例如 https://你的網域.com；測試時也加一個 http://localhost:3000。Allow credentials 不用勾。
  4. 同一頁把 Dataset 設成 Public（公開讀取），文章才能被前台讀到。
注意：Public 只代表「已發佈的文章可被讀取」，草稿不會外流；要修改／刪除仍需登入後台，很安全。
3

把 Project ID 填進首頁

告訴網站要去「哪一個」Sanity 專案抓資料。

  1. 在 sanity.io/manage 專案首頁就能看到 Project ID（一串英數字）。
  2. 用編輯器打開 index.html，搜尋 YOUR_PROJECT_ID。
  3. 把它換成你的 Project ID，存檔。dataset 通常維持 production 不用改。
// index.html 裡這一段
var SANITY = {
  projectId : '填你的 Project ID',
  dataset   : 'production',
  apiVersion: '2024-01-01'
};
填好就完成了。只要 Sanity 裡有已發佈的文章，重新整理首頁，三個方塊就會自動換成你的最新文章；還沒有文章時，會維持現在的示意卡片，畫面不會壞。
✎

怎麼開始寫部落格文章

  1. 打開你的 Sanity Studio（本機 localhost:3333 或部署後的網址）。
  2. 左側點 文章 → 右上 ＋（建立新文件）。
  3. 填 標題 →「網址代稱 slug」按旁邊的 Generate 自動產生。
  4. 選 分類標籤（例：策略思維／申論寫作／心態調整，會顯示在卡片上）。
  5. 上傳 封面圖、寫 摘要、在 內文 用所見即所得編輯器打字、插圖、加粗、做標題。
  6. 右下角按 Publish 發佈。回首頁重新整理即可看到。
  • 卡片預設只顯示 最新 3 篇（依發佈時間排序）。
  • 想改顯示幾篇：在 index.html 把查詢字串的 [0...3] 改成例如 [0...6]。
  • 想暫時隱藏某篇：在後台把它 Unpublish 即可，不必刪。
關於點進去看全文：目前卡片只顯示標題與封面，還沒有「單篇文章內頁」。等你文章寫順了、想要點卡片開全文頁，再跟我說，我幫你做文章內頁並把卡片接上連結。

任何一步卡住（終端機報錯、CORS 設定、抓不到文章），把畫面截圖回來，我針對那步給你更細的指示。
